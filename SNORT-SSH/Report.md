## ATTACK SENARIO
# Objective
detect an SSH brute force attack hapenning on an ssh server port 22

# Attack tool used 
Hydra

# Attack Behaviour Expected 
Multiple tcp connection of port 22
Repeated auth faliure on port 22 
short time interval between each login 
Same source IP with different empherial ports restarting the connection after each failed login attempt

## DETECTION LOGIC
The Rule used was simple where it checked if the incomming traffic and outgoing is tcp or not if yes then is it for this given certain ip for this port if yes then show this message

	"alert tcp any any <> 192.168.56.103 22 (msg : "Some one is trying to connect to ssh server";sid:1000000002;rev:1)"

1) traffic must be tcp
2) incoming from any with any port but must be dedicated to 192.168.56.103 this ip and for port 22
3) if these rules got triggred then show the given message
