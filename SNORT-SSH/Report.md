# Objective

Same source IP with different empherial ports restarting the connection after each failed login attempt

## DETECTION LOGIC
The rule used is easy and is not an dashboard overfloing rule
	"alert tcp any any -> 192.168.56.103 22 (flags:S ;threshold: type both , track by_src, count 5, seconds 10 ;msg : "Some one is trying to connect to ssh server";sid:1000000002;rev:1)"

1) traffic must be tcp
2) As every tcp connection starts with an SYN flag so our detection will mark that 
3) If more then 5  packet to port 22 comes with 10 seconds then our rule will get triggered
4) And it will show an alert text on the console
